# git-responder

`git-responder` iterates unresolved Git merge request/pull request comments and addresses each thread by either:

- prompting the model to make code changes and then committing/pushing those changes, or
- prompting the model to write a reviewer response to `.mitb-reply` when no changes are needed.

After a thread is addressed, the policy replies on the hosting provider and marks the thread resolved.
When a thread is picked up, the policy adds `:eye:` and `:takeout_box:` reactions on the tracked note, then adds a completion reaction when the response workflow completes (`:ballot_box_with_check:` on GitLab, `:rocket:` on GitHub).

The policy clears `.mitb-reply` before every model prompt and reads reply text from that file (falling back to terminal parsing only if the file is empty).

## Required environment variables

- `MITB_GIT_TOKEN`
- `MITB_GIT_PR_URL`
  - GitLab example: `http://localhost:55001/root/dragon-backend/-/merge_requests/1`
  - GitHub example: `https://github.com/root/dragon-backend/pull/1`

## Optional environment variables

- `MITB_VERIFICATION_COMMAND`  
  If set, changed-code threads are only committed/pushed/replied after this command passes.
- `MITB_VERIFICATION_TIMEOUT_SECONDS`  
  Timeout for `MITB_VERIFICATION_COMMAND` (default `480` seconds).
- `MITB_READ_ONLY`  
  Defaults to disabled. Set to `1`/`true` to force reply-only behavior: no commits/pushes, and replies should explain that code changes cannot be made in read-only mode when edits are requested.
- `MITB_GIT_ALLOWED_USERS`  
  Comma-delimited allowlist for reviewer usernames when filtering unresolved thread prompts.
- `MITB_GIT_PARALLEL_FILESYSTEM_LEASES`  
  Defaults to enabled on the standard (non-drip) path. Set to `0`/`false` to disable filesystem thread leases.
  When enabled, responders also switch to a dedicated local branch (`mitb/parallel/mr-<iid>/<worker-id>`) and integrate each thread by rebasing onto the desired branch before push.
- `MITB_GIT_LEASE_ROOT`  
  Root directory for filesystem lease state (default `${MITB_SHARED_ROOT}/leases`).
- `MITB_GIT_LEASE_TTL_SECONDS`  
  Lease expiration window in seconds (default `300`).
- `MITB_GIT_LEASE_HEARTBEAT_SECONDS`  
  Heartbeat refresh cadence in seconds (default `min(ttl, max(5, ttl/3))`).
- `MITB_GIT_WORKER_ID`  
  Optional stable worker identity for multi-responder coordination (autogenerated when unset).
- `MITB_GIT_DRIP_SOURCE`, `MITB_GIT_DRIP_TARGET`, `MITB_GIT_DRIP_HINT`  
  Enables the optional drip-emitter workflow. Filesystem lease coordination does not run in drip mode.
- `MITB_SHARED_ROOT`  
  Set by `mitb-host` to the absolute host path `~/.mitb/shared`. Parallel filesystem leases use `${MITB_SHARED_ROOT}/leases` by default.

## Build

From the project root:

```bash
cargo make wasm
```

Composed component output:

- `mitb-examples/target/gitlab_thread_responder.component.wasm`
